## 关键发现

1. **CID→字幕映射是概率性的**：同一 CID 每次 API 请求生成不同的 `auth_key`，路由到不同缓存节点。有些节点有正确数据，有些返回空 URL，有些指向别的视频（"串了"）。

2. **串的内容示例**（来自本会话 P26 重试过程）：
   - "你是不是很虚啊"（健康/健身）
   - "日本女友死都不穿JK"（娱乐）
   - "那五分化呢五次把全身练一遍"（健身）
   - "江南水乡种水稻呀"（农业）
   - "今天吃几碗烧烤的总结就是两个字"（美食）
   - ✅ 第6次： "未来展望中间件match化的可能性"（正确）

3. **3次重试不够**：P26 前3次全串，第6次才命中。skill 规定 5 次/分P 是合理的底线。

4. **P24 首次即中**：说明部分 CID 的命中率较高（约80-100%），而 P26 类型需更多重试。

## 验证方法

```python
verify_keywords = ["落地", "实践", "问题", "困难", "微服务", "SDK", "迁移", "ServiceMesh"]
matched = [kw for kw in verify_keywords if kw in body[0]["content"]]
```

仅检查 `body[0].content` 第一条字幕足够区分正确/错误内容。

## 结论

**严格 5 次重试 + 3 秒间隔** 足以覆盖绝大多数 B站 AI 字幕 CIDs。极少数 CID（如 P20 类型，20+次全错）也确实存在，但走完 5 次全错后再切换到 Kedou/Whisper 即可。
