# 调度器测试场景

## 通道特性
| 通道 | 耗时规则 | 说明 |
|------|---------|------|
| API | 40s/P固定 | 浏览器DevTools全路径，与视频时长无关 |
| Kedou | 35s/P固定 | Kedou云端处理，与视频时长无关 |
| Small | d/2.4 | Whisper Small模型，仅做 ≤400s 的视频 |
| Base | d/6 | Whisper Base模型，仅做 >400s 的视频 |

API/Kedou 互不干扰独立运行。Whisper内 Small 和 Base 共享CPU串行。
目标：minimax(各通道总耗时)。

---

## 场景1：全通道可用，10个P等长300s

**输入：**
- API可用, Kedou剩20, Base有, Small有
- 10个P, 每个300s
- 无running任务

**推理：**
等长则按成本：API=40, Kedou=35, Small=300/2.4=125, Base=300/6=50
Best = API3+Kedou4+Base3, 尝试分配：
- API 3个: 3×40=120s
- Kedou 4个: 4×35=140s
- Base 3个: 3×50=150s
瓶颈 = max(120, 140, 150) = **150s**

---

## 场景2：API不可用，Kedou充足，混合时长（60s~900s）

**输入：**
- API不可用, Kedou剩20, Base有, Small有
- 8个P: 60, 80, 100, 200, 400, 600, 800, 900
- 无running任务

**推理：**
按降序: 900, 800, 600, 400, 200, 100, 80, 60
长视频→Kedou(固定35s), 短视频→Whisper(短=便宜)

尝试分配方案：

**5 Kedou + 3 Whisper:**
Kedou接最长的5个(200,400,600,800,900): 5×35=175s
Whisper接最短的3个(60,80,100,均≤400→Small):
60/2.4+80/2.4+100/2.4 = 25+33+42=100s
瓶颈 = max(175, 100) = **175s** ✓

**4 Kedou + 4 Whisper:**
Kedou(400,600,800,900): 4×35=140s
Whisper(60,80,100,200,均≤400→Small):
25+33+42+83=183s
瓶颈 = max(140, 183) = 183s ✗

**6 Kedou + 2 Whisper:**
Kedou(100,200,400,600,800,900): 6×35=210s
Whisper(60,80): 25+33=58s
瓶颈 = 210s ✗

**3 Kedou + 5 Whisper:**
Kedou(600,800,900): 3×35=105s
Whisper(60,80,100,200,400): 25+33+42+83+167=350s
瓶颈 = 350s ✗

最优: **5 Kedou(175s) + 3 Whisper(100s)** → 瓶颈 **175s**

## 场景3：仅Whisper（API/Kedou均不可用）

**输入：**
- API不可用, Kedou=0, Base有, Small有
- 10个P: 60, 80, 100, 150, 200, 300, 400, 500, 700, 900
- 无running任务

**推理：**
只有Whisper路线，无可优化分配。
- ≤400s的6个(60,80,100,150,200,300,400) → Small: d/2.4
  60/2.4=25, 80/2.4=33, 100/2.4=42, 150/2.4=63, 200/2.4=83, 300/2.4=125, 400/2.4=167
  Small合计: 25+33+42+63+83+125+167 = **538s**
- >400s的3个(500,700,900) → Base: d/6
  500/6=83, 700/6=117, 900/6=150
  Base合计: 83+117+150 = **350s**
  
全部串行: 538+350 = **888s**
瓶颈 = 888s

## 场景4：中途调度（有running任务），全通道可用

**输入：**
- API可用(1), Kedou剩10, Base有, Small有
- Running:
  - Kedou: P=3, dur=200, elapsed=10 (剩余 35-10=25s)
  - Base: P=7, dur=300, elapsed=30 (剩余 300/6-30=50-30=20s)
- 新Pending: 5个P [100, 200, 400, 600, 800]

**推理：**
按降序: 800, 600, 400, 200, 100
已有忙碌: Kedou 25s, Whisper 20s

尝试配比:
**3 API + 2 Kedou:**
API: 0 + 3×40 = 120s
Kedou: 25 + 2×35 = 95s
Whisper: 20 + 0 = 20s
瓶颈 = max(120, 95, 20) = **120s** ✓

**2 API + 3 Kedou:**
API: 2×40=80s
Kedou: 25+3×35=130s
瓶颈 = 130s ✗

**4 API + 1 Kedou:**
API: 4×40=160s
Kedou: 25+35=60s
瓶颈 = 160s ✗

**全部走Whisper:**
Whisper: 20 + 100/2.4+200/2.4+400/2.4+800/6+600/6
= 20 + 42+83+167+133+100 = 545s
瓶颈 = 545s ✗

最优: **3 API(120s) + 2 Kedou(95s)** → 瓶颈 **120s**

## 场景5：全部短P（≤200s），Kedou仅剩3次，API可用

**输入：**
- API可用(1), Kedou仅剩3, Base有, Small有
- 8个P: 30, 45, 60, 80, 90, 120, 150, 200
- 无running任务

**推理：**
按降序: 200, 150, 120, 90, 80, 60, 45, 30

**约束：API可用，Kedou余量3≥8？→ 约束生效，Whisper不得为瓶颈**

**API 1 + Kedou 3 + Small 4（最优）：**
API(200): 1×40=40s
Kedou(150,120,90): 3×35=105s
Small(80,60,45,30): 33.3+25+18.8+12.5=89.6s
whisper(89.6) ≤ max(40,105)=105 ✅
瓶颈 = max(40, 105, 89.6) = **105s**

**验证违反约束的方案（旧法）：**
API 2 + Kedou 2 + Small 4: API(200,150)=80, Kedou(120,90)=70, Small(80,60,45,30)=89.6
whisper(89.6) > max(80,70)=80 ❌ 拒绝

最优: **API 1(40s) + Kedou 3(105s) + Small 4(90s)** → 瓶颈 **105s**

## 场景6：全部长P（>400s），Kedou仅剩2，API不可用

**输入：**
- API不可用, Kedou仅剩2, Base有, Small无
- 6个P: 450, 500, 600, 700, 800, 900
- 无running任务

**推理：**
All >400s → Whisper只能走Base(d/6)。
按降序: 900, 800, 700, 600, 500, 450

Kedou仅2次，必须给最长的：
Kedou 2个(900,800): 2×35=70s
Base 4个(700,600,500,450): (700+600+500+450)/6 = 2250/6 = 375s
瓶颈 = max(70, 375) = **375s**

验证：Kedou给短的？→ Kedou(450,500)=70, Base(900-600)=3000/6=500s ✗

最优: **Kedou接最长2个(70s) + Base接剩余4个(375s)** → 瓶颈 **375s**

## 场景7：混合时长含≤400s，Kedou仅剩2，API不可用

**输入：**
- API不可用, Kedou仅剩2, Base有, Small有
- 6个P: 100, 200, 300, 400, 500, 600
- 无running任务

**推理：**
按降序: 600, 500, 400, 300, 200, 100
Kedou仅2次，给最长的两个(600,500): 2×35=70s
Whisper(400,300,200,100)均≤400→Small: (400+300+200+100)/2.4=1000/2.4=417s
瓶颈 = max(70, 417) = **417s**

验证：Kedou给短的？→ Kedou(100,200)=70, Whisper(600-300)=Base(600,500)+Small(400,300)=100+83+167+125=475s ✗

最优: **Kedou接最长2个(70s) + Small接剩余4个(417s)** → 瓶颈 **417s**

瓶颈 = 160s ✗

最优: **API 3(120s) + Kedou 2(70s) + Base 1(75s)** → 瓶颈 **120s**
