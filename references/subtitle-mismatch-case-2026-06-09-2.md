## 字幕串台过程

### 第1次尝试：API+SESSDATA

```
curl "https://api.bilibili.com/x/player/v2?aid=116718907168519&cid=38986582609&subtitle=1"
-H "Cookie: SESSDATA=xxx; buvid3=xxx"
```

**返回：** `subtitles: [{subtitle_url: "//aisubtitle.hdslb.com/..."}]` — 有返回，非空！

**下载字幕JSON：** 536条中文字幕

**首条内容：** "good good li加me"
**末条内容：** "我会为您做出更多用心的作品"

**判断：** 表面看起来有内容，但"good good li加me"开头明显不像一个讲"发财女孩"故事的视频。

### 用户触发纠错

用户问：**"为啥主角一开始吃喝不看钱，后面有钱了，又开始改变消费观"**

这个问题的前提（小雨的消费观转变故事）在已获取的字幕中完全找不到对应内容。字幕讲的是"兴趣与才华"的独白，没有任何关于"女孩/小雨/发财/消费观"的词。

### 确认失败

用户说的故事与字幕内容完全对不上 → 确认AI字幕串台。

### 第2次尝试：浏览器页面验证

用 `browser_navigate` 打开B站视频页：
- **标题确认：** "我被一个靠小本生意发财的女孩，深深震撼到了"
- **简介确认：** 描述小雨的故事线
- **浏览器API再次请求：** 带登录态仍返回 `subtitles: []`（need_login_subtitle: true）

→ 确认该视频在B站没有AI字幕。之前的536条字幕来自其他视频的缓存污染。

### 第3次尝试：Kedou提取（成功）

```
Kedou URL: https://www.kedou.life/caption/subtitle/bilibili
粘贴URL: https://www.bilibili.com/video/BV1qfEL6XE7g
提取结果: 536条SRT字幕 ✅
首条内容: "小雨说"
```

Kedou成功提取，内容完全匹配标题和简介。

### 教训总结

| 方面 | 教训 |
|------|------|
| **API中可能返回536条"看起来完整"的字幕** | 不是空列表不等于正确。AI字幕API可能返回完整但内容完全串台的字幕 |
| **有SESSDATA仍可能返回空** | 即使有登录态，同时有 `subtitles: []` 和 `need_login_subtitle: true`，也不代表该视频有AI字幕可用 |
| **用户的问题是"校验器"** | 用户基于字幕内容问的具体问题是最好的验证手段。如果问题中的关键人物/情节在字幕中找不到，立刻怀疑串台 |
| **Kedou vs API的差异** | 同一视频，API串回其他视频内容，Kedou正常提取536条正确字幕。Kedou的ASR机制不受B站缓存污染影响 |
| **b23.tv/av{数字} URL** | bilibili_api.parse_link支持b23.tv/av{超长数字}格式，返回的resource_type是"av116718907168519"字符串而非BVID，需要特殊处理 |
