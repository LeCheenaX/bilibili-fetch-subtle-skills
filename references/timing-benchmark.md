# B站字幕获取时序基准（2026-06-08 实测）

## 环境

- CPU: 2 vCPU (未知架构，无GPU)
- RAM: 8GB
- OS: Linux 6.8.0

## 重试基准：AI字幕API内容命中率

`x/player/v2?aid=X&cid=Y&subtitle=1` 在 p=9 上实测10次：

| 结果 | 次数 | 占比 |
|------|------|------|
| 空URL（无字幕） | 1 | 10% |
| 错误内容（串到其他视频） | 6 | 60% |
| **正确内容** | **3** | **30%** |

单次API请求+验证平均耗时 **~0.25s**。建议5次重试。

## 通道一：AI字幕API（SESSDATA + x/player/v2）

| 指标 | p=3 (733s) | p=9 (851s) |
|------|-----------|-----------|
| API请求 (含验证) | ~0.25s | ~0.25s |
| 重试次数 | 1次 | 2次 (含验证) |
| 下载JSON | ~0.3s | ~0.3s |
| 转SRT | ~0.1s | ~0.1s |
| **总计（API纯耗时）** | **~0.65s** | **~0.9s** |
| 字幕条数 | 275 | 343 |

## 全路径：浏览器 DevTools 操作

从打开网页到字幕下载完成的完整链路：

| 步骤 | 耗时 |
|------|------|
| 打开网页 + 注入cookie + 刷新 | ~6s |
| 点击播放 + 等待视频加载 | ~9s |
| F12 → Network → 过滤"subtitle" → 找到请求 | ~7s |
| 查看protobuf响应，提取字幕信息 | ~3s |
| API重试 + 内容验证 | ~15s (5次×0.25s + 4次×3s间隔) |
| 下载JSON + 转SRT | ~1s |
| **总计（浏览器方案）** | **~41s** |

## 通道三：Whisper base (CPU) — 2026-06-08 实测（已废弃）

| 指标 | p=9 (851s) |
|------|-----------|
| 音频下载 (8.9MB .m4s) | ~3s |
| 模型加载 | ~2s |
| **转写耗时** | **146s** (2min26s) |
| 转写速度 | ~5.9× 实时速 |
| 转SRT | ~1s |
| **总计** | **~151s** |
| 字幕条数 | 336条（含分段） vs API 343条（完整句） |

> ⚠️ **此基准数据已废弃。** 此前的146s转写速度异常快，可能得益于当时CPU负载低或幸运调度。后续多次实测（2026-06-12）显示：同一CPU上813s(13:33)视频的base模型转写需要约17分钟（~1020s），实时倍率约0.8×。**5.9×实时速不是本机的可靠预期。**

## 通道三补充：Whisper small (CPU) — 2026-06-11 实测

| 指标 | P1 (340s视频) |
|------|--------------|
| 音频下载 (8.7MB .m4a) | ~2s (yt-dlp) |
| 模型加载 | ~3s |
| **转写耗时** | **>300s** (超时未完成) |
| 转写速度 | **<1.1× 实时速**（远慢于估算的2.4×） |
| 备注 | 300s超时后kill，未能产出SRT |

**⚠️ small模型实测远慢于估算值。** skill中写的"2.4× 实时速"是早期估算，实际CPU上340s视频跑了300s+仍未完成。small模型在纯CPU上的实际速度可能接近1×实时甚至更慢，不应作为≤400s视频的首选。**建议：≤400s视频优先用base（~6×实时），除非精度要求极高且时间充裕。**

| 通道 | 单P耗时 | 准确性 | 可靠性 |
